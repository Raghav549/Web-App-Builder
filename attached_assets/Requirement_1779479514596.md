# Requirements Document

## 1. Application Overview

### 1.1 Application Name
Ai Pop Cute Support

### 1.2 Application Description
A mobile-first social voting and fan support platform where users can create accounts, view Ai's profile and contest progress, vote/cheer for Ai, create posts, follow other users, send messages, and engage with the community. The platform features real authentication, database storage, real-time updates, and a complete social networking experience designed with a cute, clean, white-and-yellow theme.

### 1.3 Core Objective
Provide a polished, app-like browser experience for fans to support Ai in her cute/pop contest through voting, engagement, and community building, while offering full social networking capabilities for all users.

## 2. Users and Usage Scenarios

### 2.1 Target Users
- Ai's fans and supporters
- Contest participants and their supporters
- General users seeking social networking features
- Ai herself as content creator and profile owner

### 2.2 Core Usage Scenarios
- Fans visit to view Ai's profile, bio, progress, and vote/cheer for her
- Users create accounts to participate in voting, posting, commenting, and messaging
- Users create and share their own posts with media content
- Users follow each other, send messages, and build social connections
- Ai manages her profile, creates posts, views analytics, and interacts with supporters
- Users customize privacy settings and manage their social presence

## 4. Business Rules and Logic

### 4.1 Authentication and Authorization
- Users must create account with unique username and email
- Passwords must be hashed using bcrypt before storage
- Support login with email + password or username + password
- Create session/JWT token upon successful login
- Protect all pages under /home, /search, /studio, /notifications, /messages, /profile, /settings, /creator with authentication
- Special account aipopgirl@demo.com with password ai123456789 has creator role and accesses /creator dashboard
- Guest users can view public content but must login for interactions

### 4.2 User ID Assignment
- Every new user receives unique numeric User ID starting from 100001
- User ID is stored in database and displayed in profile/settings
- User ID can be used for search

### 4.3 Blue Verification Badge
- All user accounts automatically receive blue verification badge upon creation
- Badge is displayed beside name throughout the application
- Badge tooltip shows \"Verified profile\"

### 4.4 Vote/Cheer System
- Logged-in users can vote/cheer for Ai once per day
- Vote button stores vote in database with voterId, targetUserId (Ai's ID), voteDate, timestamp
- Prevent duplicate votes within same day using voteDate check
- Display success message: \"Your cheer has been counted!\"
- Update vote count in real-time across all pages
- Update progress meter immediately after vote
- Show heart burst animation on successful vote
- Display recent supporters list if privacy allows
- Rate limit vote API to prevent abuse
- Show countdown timer until next vote is available
- Vote count represents internal website support votes, not official MixChannel contest votes
- Display wording as \"Support Votes on this website\" or \"Cheer Votes\"

### 4.5 Progress Meter
- Default goal: 10,000 support votes (configurable by Ai in creator settings)
- Calculate percentage: currentVotes / goalVotes
- Display animated progress bar
- Show milestone badges at: 100, 500, 1,000, 5,000, 10,000 votes
- Display next milestone and votes needed
- Update live after each vote

### 4.6 Post System
- Users can create posts with text caption, image, video, or mixed media
- Support hashtags and mentions in captions
- Post visibility options: Public, Followers only, Private
- Post owner can disable comments or downloads
- Ai can create announcement posts and pin posts
- Post owner can edit or delete own posts
- Store post in database with authorId, caption, mediaUrl, mediaType, filterName, visibility, allowComments, allowDownloads, isPinned, timestamps

### 4.7 Like System
- Users can like posts
- Store like in database with userId, postId, timestamp
- Prevent duplicate likes (one like per user per post)
- Update like count in real-time
- Show animated micro-interaction on like button

### 4.8 Comment System
- Users can comment on posts if comments are enabled
- Support replies to comments (nested comments)
- Store comment in database with userId, postId, parentId (for replies), content, timestamps
- Update comment count in real-time
- Post owner and comment author can delete comments

### 4.9 Share System
- Users can share posts and profiles
- Use Web Share API on mobile to open native share sheet
- If Web Share API not available, copy link and show toast notification
- Store share in database with userId, targetType (profile/post), targetId, timestamp
- Update share count in real-time

### 4.10 Download System
- Users can download post media if post owner allows downloads
- Store download in database with userId, postId, timestamp
- Update download count in real-time
- If downloads disabled, show message: \"Download disabled by creator\"

### 4.11 View System
- Increment view count when user views profile or post
- Store view in database with viewerId (optional), targetType (profile/post), targetId, ipHash (optional), timestamp
- Update view count in real-time
- Track unique views using viewerId or ipHash

### 4.12 Follow System
- Users can follow other users
- For public profiles: follow is immediately approved
- For private profiles: follow creates pending request, owner must approve
- Store follow in database with followerId, followingId, status (approved/pending/rejected), timestamp
- Update follower/following counts in real-time
- Send notification to followed user

### 4.13 Private Profile System
- Users can make profile private in privacy settings
- Private profiles show limited information to non-followers: name, username, photo, badge
- Posts are hidden unless viewer is approved follower
- Follow button becomes \"Request Follow\" for private profiles
- Profile owner sees follow requests and can approve/reject
- Messages can be restricted based on privacy settings
- Followers/following lists can be hidden based on privacy settings

### 4.14 Chat/Messaging System
- Support one-to-one and group chats
- Implement end-to-end encryption architecture: generate client-side key pair, encrypt message content before storing, store encrypted message body in database
- Support sending text, image, file, live camera picture
- Display typing indicator in real-time when participant is typing
- Show message status: sent, delivered, seen
- Update seen status when message is viewed
- Display online/offline status and last seen for participants
- Support message requests for privacy-enabled users
- Allow message actions: reply, react, forward, delete for me, delete for everyone
- Group chats: support group name, group photo, add/remove members, admin role, leave group, group invite link

### 4.15 Notification System
- Generate notifications for: new vote/cheer, new follower, follow request, follow accepted, new like, new comment, new reply, new message, mention, post shared, profile viewed milestone, Ai new post, system alert
- Store notification in database with userId, type, title, body, relatedId, isRead, timestamp
- Display heart-shaped notification icon with unread count and pulse animation
- Update notifications in real-time
- Mark notifications as read when viewed

### 4.16 Search System
- Support search by name, username, numeric User ID, post caption, hashtag, MixChannel ID, group chat name
- Implement debounced search for performance
- Display results in tabs: Top, Users, Posts, Hashtags, Groups
- Store recent searches for user
- Show trending hashtags

### 4.17 Block and Restrict System
- Users can block other users: blocked user cannot view profile, send messages, or interact
- Users can restrict other users: restricted user has limited interaction capabilities
- Store block/restrict in database with userId, blockedId/restrictedId, timestamp
- Update block/restrict status in real-time

### 4.18 Activity Log
- Track user activities: login, logout, password change, profile edit, post created, post deleted, vote given, like, comment, follow, block, restrict, privacy change
- Store activity in database with userId, action, metadata, timestamp
- Display activity log in settings

### 4.19 Ai Creator Dashboard Access
- Only user with email aipopgirl@demo.com and role creator can access /creator routes
- Ai can edit profile, bio, create posts, view analytics, manage comments, see supporters, update progress goal

### 4.20 Feed Algorithm
- Prioritize Ai's pinned post at top of feed
- Show Ai's latest post near top
- Display posts from followed users and public posts
- Respect post visibility settings
- Implement infinite scroll with pagination

## 5. Exceptions and Edge Cases

| Scenario | Handling |
|----------|----------|
| User tries to vote multiple times in same day | Show message: \"You have already voted today. Next vote available in X hours.\" |
| User tries to interact without login | Show login modal with message: \"Please login to continue\" |
| User tries to access creator dashboard without creator role | Redirect to /home with error message |
| User tries to follow private profile | Create follow request with pending status, notify profile owner |
| User tries to message user with restricted message settings | Show message request or error based on privacy settings |
| User tries to download post with downloads disabled | Show message: \"Download disabled by creator\" |
| User tries to comment on post with comments disabled | Show message: \"Comments are disabled for this post\" |
| User tries to view private profile without being follower | Show limited profile information with \"Request Follow\" button |
| User tries to delete account | Show confirmation dialog, require password, delete all user data |
| User forgets password | Send password reset link to email, allow password reset |
| User uploads invalid file type | Show error message: \"Invalid file type. Please upload image or video.\" |
| Chat message fails to send | Show retry button, store message locally until sent |
| Network connection lost | Show offline indicator, queue actions for retry |
| User tries to edit another user's post | Show error message: \"You can only edit your own posts\" |
| User tries to access non-existent post/profile | Show 404 error page |
| Database connection fails | Show error message, retry connection |
| File upload fails | Show error message, allow retry |
| User tries to create duplicate username | Show error: \"Username already taken\" |
| User tries to create duplicate email | Show error: \"Email already registered\" |
| User enters weak password | Show error: \"Password must be at least 8 characters\" |
| Session expires | Redirect to login page with message: \"Session expired. Please login again.\" |
